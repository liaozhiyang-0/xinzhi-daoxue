# 建立时间与保持时间

## 1. 基本概念

建立时间是指触发器采样时钟有效边沿到来之前，输入数据必须保持稳定的最短时间。保持时间是指时钟有效边沿到来之后，输入数据仍需继续保持稳定的最短时间。二者是同步时序电路可靠工作的基本约束。

## 2. 核心公式或判断条件

常见建立时间约束：

```text
Tclk >= Tcq + Tcomb + Tsetup + Tskew
```

保持时间约束常关注：

```text
Tcq_min + Tcomb_min >= Thold + Tskew
```

其中 `Tcq` 为触发器时钟到输出延迟，`Tcomb` 为组合逻辑延迟，`Tsetup` 为建立时间，`Thold` 为保持时间，`Tskew` 为时钟偏斜。

## 3. 分析方法

1. 找出数据从源触发器到目标触发器的路径。
2. 计算最长路径，检查建立时间是否满足。
3. 计算最短路径，检查保持时间是否满足。
4. 若不满足，分析是否需要降低频率、优化逻辑或增加延迟。

## 4. 常见易错点

- 只检查建立时间，不检查保持时间。
- 将最大延迟和最小延迟混用。
- 忽略时钟偏斜对约束的影响。
- 误以为降低时钟频率一定能解决保持时间问题。

## 5. 典型例题

若 `Tcq = 1 ns`，`Tcomb = 6 ns`，`Tsetup = 1 ns`，忽略时钟偏斜，则时钟周期至少为 `8 ns`，最大频率约为 `125 MHz`。

## 6. 推荐复习建议

建议通过寄存器到寄存器路径训练建立时间和保持时间约束，区分最大路径用于建立时间、最小路径用于保持时间。
