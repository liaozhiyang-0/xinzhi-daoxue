# 建立时间与保持时间

## 1. 基本概念

建立时间和保持时间是同步时序电路中保证触发器正确采样数据的基本约束。建立时间要求数据在时钟有效边沿到来之前提前稳定；保持时间要求数据在时钟有效边沿到来之后继续稳定一段时间。

如果建立时间或保持时间不满足，触发器可能采样错误，严重时会进入亚稳态，导致后级逻辑不可预测。

## 2. 核心公式或判断条件

寄存器到寄存器路径的建立时间约束：

```text
Tclk >= Tcq_max + Tcomb_max + Tsetup + Tskew - Tslack_adjust
```

常见简化形式：

```text
Tclk >= Tcq_max + Tcomb_max + Tsetup
```

保持时间约束：

```text
Tcq_min + Tcomb_min >= Thold + Tskew
```

其中：

- `Tclk`：时钟周期。
- `Tcq`：源触发器时钟到输出延迟。
- `Tcomb`：组合逻辑路径延迟。
- `Tsetup`：目标触发器建立时间。
- `Thold`：目标触发器保持时间。
- `Tskew`：源触发器和目标触发器之间的时钟偏斜。

## 3. 分析方法

1. 找路径：确认数据从哪个源触发器输出，到哪个目标触发器输入。
2. 查最大延迟：使用 `Tcq_max + Tcomb_max` 检查建立时间。
3. 查最小延迟：使用 `Tcq_min + Tcomb_min` 检查保持时间。
4. 计算裕量：建立时间裕量和保持时间裕量均应不小于 0。
5. 给出修正措施：建立时间违例可降低频率或优化组合逻辑；保持时间违例通常需要增加最小路径延迟或调整时钟偏斜。

## 4. 常见易错点

- 只检查建立时间，不检查保持时间。
- 认为降低时钟频率一定能解决保持时间违例；实际上保持时间与时钟周期关系不大。
- 将最大路径延迟用于保持时间检查，或将最小路径延迟用于建立时间检查。
- 忽略时钟偏斜、时钟抖动和触发器 `Tcq`。
- 只给出“频率不满足”结论，没有说明是哪条路径和哪个约束不满足。

## 5. 典型例题

### 例题 1：建立时间检查

已知 `Tcq_max = 1 ns`，`Tcomb_max = 6 ns`，`Tsetup = 1 ns`，忽略时钟偏斜，则：

```text
Tclk >= 1 ns + 6 ns + 1 ns = 8 ns
```

最大时钟频率约为：

```text
fmax = 1 / 8 ns = 125 MHz
```

### 例题 2：保持时间检查

已知 `Tcq_min = 0.3 ns`，`Tcomb_min = 0.4 ns`，`Thold = 0.8 ns`，忽略时钟偏斜，则：

```text
Tcq_min + Tcomb_min = 0.7 ns < 0.8 ns
```

结论：保持时间不满足。降低时钟频率不能直接解决该问题，应考虑增加最小数据路径延迟或调整时钟树。

## 6. 推荐复习建议

- 建立时间看最长路径，保持时间看最短路径。
- 每道题都写出数据路径、时钟路径、约束公式和裕量。
- 将 D 触发器采样机制、亚稳态和同步时序约束放在一起复习。
- 做错题时标注违例类型，避免把所有时序问题都归为“频率太高”。
