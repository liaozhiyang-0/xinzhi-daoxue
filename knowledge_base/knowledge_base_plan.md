# 知识库建设计划

## 1. 知识库建设目标

面向电子信息课程群与集成电路专业方向课程群，建设可供星辰 Agent 检索、问答、错题诊断和学习规划使用的结构化课程知识库。当前阶段重点建立稳定目录体系和文档模板，具体课程知识内容后续分批补充。

## 2. 知识库组织方式

```text
课程群
→ 课程
→ 章节
→ 知识单元
→ 典型题型
→ 错题模式
→ 学习路径
```

每门课程统一包含：

- `README.md`：课程入口说明；
- `00_course_overview.md`：课程总览；
- `chapters/`：章节级知识文档；
- `questions/`：典型题型与标准解析；
- `wrong_cases/`：错题模式与诊断样例。

## 3. 课程群分类

- 电子信息公共主干课程：电路理论、模拟电子技术、数字电子技术、信号与系统、数字信号处理、通信原理、高频电子线路、电磁场与电磁波、信息论与编码、嵌入式系统。
- 集成电路专业核心课程：半导体物理、半导体器件、制造工艺、CMOS 数字集成电路、模拟 IC、数字 IC、版图、测试、封装、EDA、硬件描述语言、FPGA、SoC、射频 IC、功率半导体与功率 IC。
- 工程实践拓展课程：面向硬件描述、FPGA、嵌入式和工程实现的课程内容，可服务项目演示和工程实践能力建设。

## 4. 课程内容建设流程

1. 建立课程总览和课程定位。
2. 拆分章节目录和知识单元。
3. 为每个知识单元补充概念、公式、流程和易错点。
4. 整理典型题型、标准答案和解析。
5. 整理错题模式、错误类型和诊断建议。
6. 上传星辰 Agent 知识库并进行问答测试。
7. 根据测试结果更新资料粒度和命名规范。

## 5. Agent 使用方式

- 课程问答 Agent：优先检索课程总览、章节知识单元和典型题型。
- 错题诊断 Agent：优先检索错题模式、错误类型和标准解析。
- 学习规划 Agent：结合课程结构、错题记录和知识点优先级生成学习路径。
- 教师分析 Agent：基于课程目录和统计数据生成教学建议。

## 6. MaaS 后续微调数据沉淀方式

后续可从问答记录、错题诊断记录和教师分析记录中筛选高质量样例，脱敏后整理为 JSONL 微调数据。数据沉淀应保留课程、章节、知识点、题型、错误类型和适用场景标签，避免直接使用真实学生隐私信息。
