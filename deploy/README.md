# 部署说明

## 第一版部署目标

第一版使用 Docker Compose 描述 frontend、backend、mysql、redis、minio 和 nginx 的部署关系。当前文件为基础模板，镜像和启动命令需在真实前后端工程创建后补齐。

## 配置来源

部署环境变量参考根目录 `.env.example`。生产环境应通过安全的环境变量或密钥管理系统注入，不提交真实 `.env`。

## 验证顺序

1. 检查 `.env` 是否仅存在于本地且未提交 Git。
2. 检查数据库、缓存和对象存储容器是否启动。
3. 启动后端并验证健康检查接口。
4. 启动前端并验证页面访问。
5. 通过 Nginx 验证反向代理路径。
