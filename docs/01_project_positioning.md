# 项目定位

## 项目背景

电子信息课程群覆盖模拟电子技术、数字电子技术、集成电路基础、信号与系统等多门课程，知识点之间关联紧密，公式、判断条件和典型电路分析方法较多。传统教学中，学生常见问题集中在概念混淆、步骤缺失、公式适用条件不清和错题复盘不足。教师端也需要更低成本地汇总学生问题与薄弱环节。

## 项目定位

芯智导学定位为电子信息课程群垂类大模型系统，通过自研业务系统管理课程、题库、学习记录和统计数据，通过星辰 Agent 提供问答、诊断、规划和分析能力，通过星火 MaaS 支持后续垂类数据微调。

## 目标用户

- 学生：需要课程答疑、错题诊断和阶段性复习建议的学习者。
- 教师：需要掌握班级学习情况、问题分布和教学改进建议的课程负责人。
- 院系管理者：需要了解课程群建设成效和教学数据沉淀情况的管理人员。

## 核心场景

- 学生围绕知识点发起智能问答。
- 学生提交错题，系统给出错误类型和补救建议。
- 系统根据学习记录生成个性化学习建议。
- 教师查看班级高频问题、薄弱知识点和错题类型分布。

## 项目价值

项目将课程资料、题库、错题和学情数据统一沉淀，为课程群智能化教学提供可复用的数据基础和应用闭环。

## ToC 学生端价值

- 降低课后答疑等待成本。
- 帮助学生定位错误原因，而不仅是获得标准答案。
- 根据薄弱知识点形成更明确的复习路径。

## ToB 教师端与院系端价值

- 帮助教师识别共性问题与教学难点。
- 为课堂复盘、作业讲评和教学改革提供数据依据。
- 形成可持续扩展的课程群知识库与教学数据资产。
