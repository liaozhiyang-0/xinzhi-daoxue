# 团队任务计划

## 分工原则

第一版以演示闭环为目标，团队成员围绕产品架构、课程资料、Agent、前端、后端数据和 MaaS 优化协同推进。当前负责人均为待定，后续根据团队成员能力和时间安排确认。

| 角色 | 负责人 | 主要任务 |
| --- | --- | --- |
| 产品架构负责人 | 负责人待定 | 明确项目范围、演示链路、页面结构和答辩叙事 |
| 课程资料负责人 | 负责人待定 | 整理课程知识点、题库、错题样例和知识库材料 |
| 星辰 Agent 负责人 | 负责人待定 | 设计并调试识图解题、课程问答、学习规划和教师分析 Agent |
| 前端负责人 | 负责人待定 | 实现学生端题图上传、问答页面、教师端看板和图表展示 |
| 后端与数据负责人 | 负责人待定 | 实现 API、数据库、统计逻辑和 Agent 调用封装 |
| 星火 MaaS 与模型优化负责人 | 负责人待定 | 准备图文解题评估样例、批量推理方案和后续模型优化计划 |

## 协作节奏

- 每周同步一次文档、数据和代码进展。
- 每次功能提交需说明变更内容、运行方式和验证结果。
- 演示前完成一次全链路彩排和资料检查。
