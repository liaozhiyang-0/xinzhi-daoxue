# 技术路线

## 总体技术路线

项目采用自研业务系统承载业务流程，使用星辰 Agent 提供智能问答和分析能力，使用星火 MaaS 支持后续模型微调。前端负责清晰展示学生和教师的核心工作流，后端负责数据沉淀、权限边界、接口编排和统计分析。

## 第一阶段路线

- 建立课程资料、题库和错题样例的 Markdown / JSONL 基础资产。
- 完成学生端和教师端页面原型。
- 完成 API、数据库和 Agent 调用设计。
- 构建 Docker Compose 部署模板。
- 验证第一版演示链路。

## 第二阶段路线

- 接入真实星辰 Agent API。
- 扩充课程知识库与题库。
- 引入更细粒度的学习画像和教学统计。
- 构建 MaaS 微调数据集，进行垂类模型评估和优化。
- 支持更多课程、更多角色和更完整的运营能力。

## 星辰 Agent 使用方式

星辰 Agent 用于封装智能问答、错题诊断、学习规划和教师分析任务。自研后端负责组织请求上下文，将课程、知识点、题目、学生作答和历史记录传入 Agent，并将结构化输出返回前端展示。

## 星火 MaaS 使用方式

星火 MaaS 用于承载后续微调、评估和模型优化。第一版只准备样例 JSONL、数据格式说明和评估方案，不直接提交真实密钥或训练任务。

## 自研系统负责内容

- 用户、课程、知识点、题库和记录管理。
- Agent API 调用鉴权、重试、日志和异常处理。
- 学情统计、错题统计和学习画像计算。
- 前端页面、接口协议、数据存储和部署运维。
