# 技术路线

## 总体技术路线

项目采用自研业务系统承载业务流程，使用星辰 Agent 提供识图解题、课程问答和分析能力，使用星火 MaaS 支持图文解题评估、批量推理和后续可选微调。前端负责清晰展示学生和教师的核心工作流，后端负责数据沉淀、权限边界、接口编排和统计分析。

## 第一阶段路线

- 建立课程资料、题库、错因标签和图文解题样例的 Markdown / JSONL 基础资产。
- 完成学生端和教师端页面原型。
- 完成 API、数据库和 Agent 调用设计。
- 构建 Docker Compose 部署模板。
- 验证识图解题、课程问答、学习建议和教师分析的第一版演示链路。

## 第二阶段路线

- 接入真实星辰 Agent API。
- 扩充课程知识库与题库。
- 引入更细粒度的学习画像和教学统计。
- 构建 MaaS 图文解题评估集，进行批量推理、模型评估和后续可选微调。
- 支持更多课程、更多角色和更完整的运营能力。

## 星辰 Agent 使用方式

星辰 Agent 用于封装识图解题、课程问答、学习规划和教师分析任务。复杂题图由识图解题 Agent 处理；概念解释、公式条件和简单纠错由课程问答 Agent 处理；错题诊断不再作为独立核心 Agent。自研后端负责组织请求上下文，将课程、知识点、题图、学生追问和历史记录传入 Agent，并将结构化输出返回前端展示。

## 星火 MaaS 使用方式

星火 MaaS 用于承载图文解题样例构造、批量推理、评估和后续可选模型优化。第一版只准备样例 JSONL、Prompt、数据格式说明和评估方案，不直接提交真实密钥或训练任务。

## 自研系统负责内容

- 用户、课程、知识点、题库和记录管理。
- Agent API 调用鉴权、重试、日志和异常处理。
- 学情统计、错因提示统计、题图解题统计和学习画像计算。
- 前端页面、接口协议、数据存储和部署运维。
